## What moved

1. **`n-plus-one` 4× faster.** Run 1 spawned performance-engineer for 303s of
   doomed dynamic verification; run 2 stayed in the main thread and answered
   statically. Biggest single win of the release.
2. **`policy` backend stage 201s → 88s** — leaner brief-following after the
   body rewrites; overall case still dominated by ~400s of main-thread
   orchestration (unchanged lever).
3. **`action` regressed to timeout.** backend-developer took 378s (vs 141s) —
   the richer canon (tests-first, attributes, guard clauses) produced a bigger,
   better diff, and the qa/tech-lead stages ran past the 1200s cap. Checks
   passed because artifacts landed first. Levers: raise `EVAL_TIMEOUT` for
   multi-stage write cases, and the qa scope rule (below).
4. **qa-engineer is still the hog and got hungrier**: 448s / 108.6k tokens
   writing more tests than the brief asked (up from 82.8k). Confirmed the
   scope-creep hypothesis from run 1.
